## Description

[Nest](https://github.com/nestjs/nest) framework TypeScript starter repository.

## Project setup

```bash
$ npm install
```

## Compile and run the project

```bash
# development
$ npm run start

# watch mode
$ npm run start:dev

# production mode
$ npm run start:prod
```

## Run tests

```bash
# unit tests
$ npm run test

# e2e tests
$ npm run test:e2e

# test coverage
$ npm run test:cov
```

```bash
$ npm install -g @nestjs/mau
$ mau deploy
```

With Mau, you can deploy your application in just a few clicks, allowing you to focus on building features rather than managing infrastructure.

## Fastify
- Đã cấu hình Fastify làm HTTP adapter trong `main.ts`.

## Slonik
- Đã cấu hình Slonik connection pool toàn cục qua `SlonikModule`.
- Sử dụng biến môi trường `DATABASE_URL` để cấu hình kết nối PostgreSQL.

## Docker Compose

### Chạy toàn bộ hệ thống (API + Database + Redis)
```bash
# Build và chạy tất cả services
docker-compose up -d

# Xem logs
docker-compose logs -f

# Dừng tất cả services
docker-compose down
```

### Chạy chỉ database và Redis
```bash
docker-compose up -d postgres redis
```

### Chạy development mode
```bash
# Chạy database và Redis
docker-compose up -d postgres redis

# Chạy API trong development mode
npm run start:dev
```

## Environment Variables

Tạo file `.env` trong root directory:
```env
NODE_ENV=development
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/nestjs_db
REDIS_URL=redis://localhost:6379
PORT=4000
```

## Ví dụ sử dụng Slonik trong service
```ts
import { Inject, Injectable } from '@nestjs/common';
import { SLONIK_POOL } from './database/slonik/slonik.module';
import { DatabasePool } from 'slonik';

@Injectable()
export class UserService {
  constructor(
    @Inject(SLONIK_POOL) private readonly pool: DatabasePool,
  ) {}

  async getUsers() {
    return this.pool.any(sql`SELECT * FROM users`);
  }
}
```

## Chạy dự án

### Development
```bash
npm run start:dev
```

### Production
```bash
npm run build
npm run start:prod
```

## Mở rộng
- Tạo các module/service mới và inject `SLONIK_POOL` để truy cập database.
- Thêm các DTO trong từng module tương ứng.
- Sử dụng các shared components (interceptors, guards, pipes) trong modules.
